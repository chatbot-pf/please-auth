# Waitlist Plugin

The `@please-auth/waitlist` plugin intercepts **all** Better Auth registration paths — email/password, OAuth, magic-link, OTP, anonymous, and more — and gates sign-ups behind an invite-based waitlist system.

## How It Works

1. A visitor calls `waitlist.join({ email })` to join the waitlist.
2. An admin approves the entry (single or bulk).
3. The approved user receives an invite code (if `requireInviteCode` is enabled).
4. The user signs up with the invite code — the plugin verifies it automatically.
5. After successful sign-up, the waitlist entry is marked as `registered`.

The plugin uses **two layers** of protection:
- **Request hooks** intercept registration endpoints before they execute.
- **Database hooks** prevent user creation even for auth methods that bypass request-level checks (e.g., OAuth callbacks).

## Server Setup

```ts
import { betterAuth } from "better-auth";
import { waitlist } from "@please-auth/waitlist";

export const auth = betterAuth({
  plugins: [
    waitlist({
      requireInviteCode: true,
      inviteCodeExpiration: 172800, // 48 hours (default)
      adminRoles: ["admin"],
      sendInviteEmail: async ({ email, inviteCode, expiresAt }) => {
        await sendEmail({
          to: email,
          subject: "You're invited!",
          body: `Your invite code: ${inviteCode}`,
        });
      },
      onJoinWaitlist: async (entry) => {
        console.log(`New waitlist entry: ${entry.email}`);
      },
      onApproved: async (entry) => {
        console.log(`Approved: ${entry.email}`);
      },
    }),
  ],
});
```

## Client Setup

```ts
import { createAuthClient } from "better-auth/client";
import { waitlistClient } from "@please-auth/waitlist/client";

const auth = createAuthClient({
  plugins: [waitlistClient()],
});
```

## Options

### `enabled`
- **Type:** `boolean`
- **Default:** `true`

Whether the waitlist gate is active. Set to `false` to disable all waitlist checks without removing the plugin.

### `requireInviteCode`
- **Type:** `boolean`
- **Default:** `false`

When `true`, users must provide a valid invite code during registration. The code can be passed in the request body as `inviteCode` or in the `x-invite-code` header.

### `inviteCodeExpiration`
- **Type:** `number`
- **Default:** `172800` (48 hours)

Invite code TTL in seconds. After this period, the invite code expires and cannot be used for registration.

### `maxWaitlistSize`
- **Type:** `number | undefined`
- **Default:** `undefined` (no limit)

Maximum number of entries allowed on the waitlist. When the limit is reached, new `join` requests return a `WAITLIST_FULL` error.

### `skipAnonymous`
- **Type:** `boolean`
- **Default:** `false`

When `true`, anonymous sign-ins bypass the waitlist check.

### `autoApprove`
- **Type:** `boolean | ((email: string) => boolean | Promise<boolean>)`
- **Default:** `undefined`

Automatically approve entries when they join. Pass `true` to auto-approve all, or a function for conditional logic:

```ts
waitlist({
  autoApprove: (email) => email.endsWith("@mycompany.com"),
});
```

### `interceptPaths`
- **Type:** `string[]`
- **Default:** Built-in list covering all standard registration paths

Override the list of Better Auth paths that the plugin intercepts. Defaults include:

- `/sign-up/email`
- `/callback/`
- `/oauth2/callback/`
- `/magic-link/verify`
- `/sign-in/email-otp`
- `/email-otp/verify-email`
- `/phone-number/verify`
- `/sign-in/anonymous`
- `/one-tap/callback`
- `/siwe/verify`

### `adminRoles`
- **Type:** `string[]`
- **Default:** `["admin"]`

Roles that are allowed to perform admin actions (approve, reject, bulk-approve, list, stats).

### `sendInviteEmail`
- **Type:** `(data: { email: string; inviteCode: string; expiresAt: Date }) => void | Promise<void>`

Called when an entry is approved. Implement this to deliver invite codes to users.

### `onJoinWaitlist`
- **Type:** `(entry: WaitlistEntry) => void | Promise<void>`

Called after an entry joins the waitlist.

### `onApproved`
- **Type:** `(entry: WaitlistEntry) => void | Promise<void>`

Called after an entry is approved.

### `onRejected`
- **Type:** `(entry: WaitlistEntry) => void | Promise<void>`

Called after an entry is rejected.

### `schema`
- **Type:** `{ waitlist?: { modelName?: string; fields?: Record<string, string> } }`

Customize table and field names for the waitlist schema.

## API Endpoints

### Public Endpoints

#### `POST /waitlist/join`

Join the waitlist.

**Body:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `email` | `string` | Yes | Email address |
| `referredBy` | `string` | No | Referral source |
| `metadata` | `Record<string, unknown>` | No | Arbitrary metadata |

**Response:**
```json
{
  "id": "abc123",
  "email": "user@example.com",
  "status": "pending",
  "lookupToken": "uuid-token",
  "createdAt": "2025-01-01T00:00:00.000Z"
}
```

#### `GET /waitlist/status?token=<lookupToken>`

Check waitlist status using a lookup token.

**Response:**
```json
{
  "status": "pending"
}
```

#### `POST /waitlist/verify-invite`

Verify an invite code before attempting registration.

**Body:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `inviteCode` | `string` | Yes | The invite code to verify |

**Response:**
```json
{
  "valid": true,
  "email": "user@example.com"
}
```

### Admin Endpoints

All admin endpoints require an authenticated session with a role in `adminRoles`.

#### `POST /waitlist/approve`

Approve a single waitlist entry.

**Body:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `email` | `string` | Yes | Email to approve |

#### `POST /waitlist/reject`

Reject a single waitlist entry.

**Body:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `email` | `string` | Yes | Email to reject |

#### `POST /waitlist/bulk-approve`

Bulk approve entries by email list or count (FIFO from oldest pending).

**Body:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `emails` | `string[]` | No | Specific emails to approve |
| `count` | `number` | No | Number of oldest pending entries to approve |

**Response:**
```json
{
  "approved": 5,
  "entries": [...]
}
```

#### `GET /waitlist/list`

List waitlist entries with pagination.

**Query Parameters:**
| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `status` | `"pending" \| "approved" \| "rejected" \| "registered"` | — | Filter by status |
| `page` | `number` | `1` | Page number |
| `limit` | `number` | `20` | Items per page (max 100) |
| `sortBy` | `"createdAt" \| "email" \| "status"` | `"createdAt"` | Sort field |
| `sortDirection` | `"asc" \| "desc"` | `"desc"` | Sort direction |

#### `GET /waitlist/stats`

Get waitlist statistics.

**Response:**
```json
{
  "total": 100,
  "pending": 60,
  "approved": 25,
  "rejected": 5,
  "registered": 10
}
```

## Error Codes

Use `WAITLIST_ERROR_CODES` to match errors programmatically:

```ts
import { WAITLIST_ERROR_CODES } from "@please-auth/waitlist";

if (error.code === WAITLIST_ERROR_CODES.NOT_APPROVED.code) {
  // redirect to waitlist page
}
```

| Code | Message |
|------|---------|
| `EMAIL_ALREADY_IN_WAITLIST` | This email is already on the waitlist |
| `WAITLIST_ENTRY_NOT_FOUND` | Waitlist entry not found |
| `NOT_APPROVED` | You must be approved from the waitlist to register |
| `INVALID_INVITE_CODE` | Invalid or expired invite code |
| `INVITE_CODE_REQUIRED` | An invite code is required to register |
| `ALREADY_REGISTERED` | This waitlist entry has already been used for registration |
| `WAITLIST_FULL` | The waitlist is currently full |
| `UNAUTHORIZED_ADMIN_ACTION` | You are not authorized to perform this action |

## Database Schema

The plugin creates a `waitlist` table with the following fields:

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Primary key |
| `email` | `string` | Email address (unique) |
| `status` | `string` | `pending`, `approved`, `rejected`, or `registered` |
| `lookupToken` | `string` | Token for status lookups (unique) |
| `inviteCode` | `string?` | Invite code (unique, nullable) |
| `inviteExpiresAt` | `date?` | Invite code expiration |
| `referredBy` | `string?` | Referral source |
| `metadata` | `string?` | JSON-encoded metadata |
| `approvedAt` | `date?` | Approval timestamp |
| `rejectedAt` | `date?` | Rejection timestamp |
| `registeredAt` | `date?` | Registration timestamp |
| `createdAt` | `date` | Creation timestamp |
| `updatedAt` | `date` | Last update timestamp |

## Status Flow

```
pending → approved → registered
    ↘
     rejected
```
