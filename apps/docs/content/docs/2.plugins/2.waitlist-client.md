# Waitlist Client

The `@please-auth/waitlist/client` module provides typed client helpers and reactive atoms for interacting with the waitlist plugin.

## Setup

```ts
import { createAuthClient } from "better-auth/client";
import { waitlistClient } from "@please-auth/waitlist/client";

const auth = createAuthClient({
  plugins: [waitlistClient()],
});
```

## Client Methods

### `waitlist.join`

Join the waitlist.

```ts
const { data, error } = await auth.waitlist.join({
  email: "user@example.com",
  referredBy: "friend-123",       // optional
  metadata: { source: "landing" }, // optional
});

// data: { id, email, status, lookupToken, createdAt }
```

### `waitlist.status`

Check the waitlist status using the lookup token returned from `join`.

```ts
const { data } = await auth.waitlist.status({
  token: "lookup-token-uuid",
});

// data: { status: "pending" | "approved" | "rejected" | "registered" }
```

### `waitlist.verifyInvite`

Verify an invite code before attempting registration.

```ts
const { data } = await auth.waitlist.verifyInvite({
  inviteCode: "invite-code-uuid",
});

// data: { valid: boolean, email: string | null }
```

## Reactive Atoms

The client plugin exposes a `waitlistStats` atom (via [nanostores](https://github.com/nanostores/nanostores)) that automatically refreshes when admin actions are performed.

```ts
// Access the reactive stats atom
const stats = auth.waitlistStats;

// In a framework (e.g., React with @nanostores/react)
import { useStore } from "@nanostores/react";

function WaitlistDashboard() {
  const { data, isPending } = useStore(stats);

  if (isPending) return <p>Loading...</p>;

  return (
    <div>
      <p>Total: {data.total}</p>
      <p>Pending: {data.pending}</p>
      <p>Approved: {data.approved}</p>
    </div>
  );
}
```

The stats atom refreshes automatically after:
- `waitlist.join` calls
- Admin approve/reject/bulk-approve actions

## Sign-up with Invite Code

When `requireInviteCode` is enabled on the server, pass the invite code during sign-up:

```ts
// Via request body
await auth.signUp.email({
  email: "user@example.com",
  password: "secure-password",
  inviteCode: "invite-code-uuid",
});

// Or via header
await auth.signUp.email(
  { email: "user@example.com", password: "secure-password" },
  { headers: { "x-invite-code": "invite-code-uuid" } },
);
```

## Complete Example

```ts
// 1. Join the waitlist
const { data: joinResult } = await auth.waitlist.join({
  email: "user@example.com",
});

// 2. Save the lookup token (e.g., in localStorage)
localStorage.setItem("waitlistToken", joinResult.lookupToken);

// 3. Later, check status
const { data: statusResult } = await auth.waitlist.status({
  token: localStorage.getItem("waitlistToken")!,
});

if (statusResult.status === "approved") {
  // 4. Verify the invite code (from email)
  const { data: verification } = await auth.waitlist.verifyInvite({
    inviteCode: inviteCodeFromEmail,
  });

  if (verification.valid) {
    // 5. Complete registration
    await auth.signUp.email({
      email: "user@example.com",
      password: "secure-password",
      inviteCode: inviteCodeFromEmail,
    });
  }
}
```
